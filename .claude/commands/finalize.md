Проаналізуй поточну розмову і знайди все що варто зафіксувати в `BUGS_AND_DECISIONS.md`.

## Що шукати в розмові

**Баги** — будь-яка помилка агента або коду, яку ми знайшли і виправили:
- Неправильний розрахунок ціни
- Агент відповів неправильно (не те сказав, не туди відправив)
- Технічна помилка (500, 400, виключення в коді)

**Рішення** — архітектурні або продуктові рішення що прийняли:
- Нова інструкція в промпті
- Зміна логіки розрахунку
- Нове правило поведінки агента

**Уроки** — що дізнались, що треба пам'ятати на майбутнє:
- Де шукати ціни (Google Sheets KB, не в коді)
- Як перевіряти діалоги (через БД)
- Що є стандартним, а що нестандартним

## Формат запису

Для **кожного знайденого пункту** додай запис у відповідний розділ `BUGS_AND_DECISIONS.md`:

### Баг:
```
### N. [Коротка назва бага]
**Симптом:** що спостерігали
**Причина:** чому сталось
**Фікс:** що змінили і де (файл/рядок)
**Дата:** YYYY-MM-DD
```

### Рішення:
```
### N. [Назва рішення]
**Рішення:** що вирішили робити
**Причина:** навіщо
**Альтернатива яку відкинули:** (якщо є)
```

### Урок:
```
### N. [Назва уроку]
**Урок:** що запам'ятати
**Контекст:** звідки це прийшло
```

## Дії

1. Прочитай `BUGS_AND_DECISIONS.md` щоб знати поточну нумерацію
2. Знайди в розмові все нове що ще не зафіксовано
3. Допиши нові записи в кінець відповідного розділу
4. Онови дату "Останнє оновлення" у шапці файлу
5. Закомить: `git add BUGS_AND_DECISIONS.md && git commit -m "docs: finalize session — [короткий опис що додали]"`
